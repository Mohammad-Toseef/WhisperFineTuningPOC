## Key Findings (from codebase + data audit)
- **Will current LoRA learn domain terms?** Not with the smoke-test config. `q_proj/v_proj`
  (attention) only adapts accent/pacing. Learning vocabulary + the izafat ( ِ ) mark needs
  the **MLP layers (`fc1/fc2`)** and output projection, which were frozen.
- **Label consistency is already good** — every occurrence of the four spiritual terms carried
  the izafat mark; **frequency** (each appears only a handful of times) is the real bottleneck.
- **The root reviewed manifests are NOT new data** — `all_batches`, `Batch-1 5-july`, and
  `Test Batch` all describe the **same EP1–EP10** videos, re-chunked. ≈10 episodes / ~3 hrs total.
  `Oracle-Sayings.json` is text-only poetry (different pipeline).
- **On disk** (after user deleted the duplicate `Batch1_EP10`): `Batch1_EP23/audio` = EP1–23,
  `Batch2_EP24_EP50/audio` = EP24–50 (EP39 missing). → **49-ep run = EP1–EP50 minus EP39**,
  spanning two folders.
- **Code layout:** no `src/train.py`/`src/evaluate.py` — all inline in `modal_app.py`. Train/eval
  split is created at build time in `src/dataset_builder.py` (was random per-clip 90/10, seed 42,
  no episode awareness → leakage). LoRA `target_modules` and `max_steps` are pure YAML reads.
